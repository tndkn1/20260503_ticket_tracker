# スクリーンショット付き操作手順書

## はじめに
本手順書は、ITSM インシデント管理システムの主要画面・操作を図解で説明します。  
画面レイアウトおよび表示内容の実例を想定した解説です。

---

## 3. インシデント起票ダイアログ

```
┌──────────────────────────────────┐
│ インシデント起票                  │
├──────────────────────────────────┤
│                                  │
│ タイトル *                        │
│ [________________________]        │
│                                  │
│ 説明 *                            │
│ ┌────────────────────────────┐   │
│ │ システムが応答しない        │   │
│ │ 午前9時ごろから...          │   │
│ │                            │   │
│ └────────────────────────────┘   │
│                                  │
│ 優先度                            │
│ [ ▼ P3 ]                         │
│                                  │
│ 担当者                            │
│ [ ▼ 未指定 ]                     │
│                                  │
│ [   キャンセル  ] [   起票   ]   │
│                                  │
└──────────────────────────────────┘
```

### 入力項目説明

| 項目 | 必須 | 説明 | 例 |
|-----|:---:|-----|-----|
| **タイトル** | ✓ | インシデント概要（1行） | `ログインシステムが応答しない` |
| **説明** | ✓ | 詳細、再現手順、影響範囲 | `午前9時ごろからログインが失敗するようになりました` |
| **優先度** | ✗ | P1～P4（デフォルト P3） | P1, P2, P3, P4 |
| **担当者** | ✗ | 対応を担当するユーザー | operator_001, admin |

### 操作手順

| ステップ | 操作 |
|----------|-----|
| 1 | **タイトル** に概要を入力 |
| 2 | **説明** に詳細を入力 |
| 3 | **優先度** ドロップダウンで優先度選択（任意） |
| 4 | **担当者** ドロップダウンで担当者選択（任意） |
| 5 | **起票** ボタンをクリック |
| → | インシデント詳細画面に遷移、Slack 通知が送信 |


### 画面構成説明

| セクション | 説明 |
|-----------|-----|
| **ヘッダー** | ID、優先度、ステータス、SLA状態を表示 |
| **基本情報** | タイトル、起票者、起票日時を表示 |
| **SLA詳細** | 応答・解決の期限と実績を表示 |
| **操作パネル** | ステータス・優先度・担当者・タイトルを変更 |
| **タイムライン** | 変更履歴・コメントを新しい順で表示 |

### バッジの意味

| バッジ | 意味 |
|-------|-----|
| `🔴 P1` | 最高優先度（Critical） |
| `[新規]` | ステータスが「new」 |
| `⚠️ 要注意` | SLA応答違反 |
| `期限超過 -45m` | SLA解決期限を45分超過 |

### 主な操作

| 操作 | 方法 |
|-----|-----|
| **ステータス変更** | ステータスボタン ▼ をクリック、新しいステータスを選択 |
| **優先度変更** | 優先度ボタン ▼ をクリック、新しい優先度を選択 |
| **担当者変更** | 担当者ボタン ▼ をクリック、新しい担当者を選択 |
| **タイトル編集** | [編集] をクリック、テキストを修正、Enter で保存 |
| **コメント追加** | 入力欄にテキスト入力、[コメント送信] をクリック |

## 5. ステータス遷移フロー

```
                          ┌─→ closed
                          │
    new ─→ assigned ─→ in_progress
     │        │              │
     │        └─────┬────────┘
     │              ↓
     │          resolved ─→ closed
     │              ↑
     └──────────────┘
```

### 遷移可能パターン

| 現在のステータス | 遷移可能な次の状態 |
|-----------------|-----------------|
| **new** | assigned, in_progress, closed |
| **assigned** | in_progress, resolved, closed |
| **in_progress** | resolved, closed |
| **resolved** | closed, in_progress（再開の場合） |
| **closed** | （遷移不可） |

## 6. SLA バッジの見方

### SLA 応答バッジ

```
┌──────────────────────────────────────────┐
│ 残り時間          │ 色   │ 説明           │
├──────────────────────────────────────────┤
│ 「3h 20m」       │ 🟢 │ 十分な余裕     │
│ 「15m」          │ 🟠 │ 警告ライン     │
│ 「-30m」         │ 🔴 │ 既に超過       │
│ 「応答済」       │ 🔵 │ 既に対応開始   │
└──────────────────────────────────────────┘
```

### 更新タイミング
- バッジの残り時間は **60秒ごと**に自動更新
- ページ手動リロード時にも即座に更新

## 7. インシデント一覧フィルタ

### キーワード検索の使い方

```
検索ボックス: [____________________] (🔍)

対象フィールド:
✓ タイトル
✓ 説明
✓ 起票者
```

**例：**
- `ログイン` → タイトルまたは説明に「ログイン」を含むインシデント表示
- `admin` → 起票者が「admin」のインシデント表示

### 優先度フィルタの使い方

```
[ すべて ] [ P1 ] [ P2 ] [ P3 ] [ P4 ]
```

1 つのボタンをクリックすると、その優先度のみ表示

### ステータスボタンの使い方

```
[ new ] [ assigned ] [ in_progress ] [ resolved ] [ closed ]
```

複数クリック可能（OR 検索）。クリックしたボタンのいずれかに該当するインシデントが表示

## 9. よくある操作シーン

### シーン 1: 新しいインシデントを起票する

```
1. ダッシュボードを開く
2. [+ インシデント起票] をクリック
3. タイトル、説明を入力
4. 優先度・担当者を選択（任意）
5. [起票] をクリック
6. 詳細画面が表示、自動で ID 採番
7. Slack に通知
```

### シーン 2: インシデントの対応を開始する

```
1. インシデント詳細画面を開く
2. [ステータス ▼] をクリック
3. 「assigned」または「in_progress」を選択
4. タイムラインに記録、Slack に通知
```

### シーン 3: インシデントを解決する

```
1. インシデント詳細画面を開く
2. [ステータス ▼] をクリック
3. 「resolved」を選択
4. SLA 達成判定（期限内であれば「SLA 達成」表示）
5. Slack に通知
```

### シーン 4: コメントを追加する

```
1. インシデント詳細画面を開く
2. 下部のコメント入力欄にテキスト入力
3. [コメント送信] をクリック
4. タイムラインに新しいエントリが追加される
```
