# 解説動画スクリプト

現在の実装に合わせた短時間デモ用スクリプトです。想定尺は 8 から 10 分です。

## シーン 1: 概要（45秒）

### 画面

トップページまたはタイトルスライド。

### ナレーション

この動画では、ITSM Incident Tracker の基本操作を紹介します。このシステムでは、インシデントの起票、担当者設定、ステータス更新、SLA の確認、監査ログの確認をひとつの画面フローで行えます。

主な機能は、パスワードまたは GitHub によるログイン、優先度別 SLA、Slack 通知、admin による論理削除です。

## シーン 2: ログイン（1分）

### 実演

1. `/login` を開く。
2. ユーザー名とパスワードを入力する。
3. `ログイン` を押す。
4. ログアウト後、`GitHub でログイン` ボタンも示す。

### ナレーション

ログイン方法は 2 種類あります。通常はユーザー名とパスワードでログインします。GitHub OAuth が設定されている環境では、GitHub アカウントでもログインできます。初回の GitHub ログインでは member ユーザーが自動作成され、メールアドレスが既存ユーザーと一致する場合は自動でリンクされます。

## シーン 3: ダッシュボード（1分30秒）

### 実演

1. KPI カードを順番に指す。
2. ステータスボタンをクリックして一覧を絞り込む。
3. 検索ボックスと優先度フィルタを操作する。
4. 更新アイコンで再取得する。

### ナレーション

トップページでは、総インシデント数、未解決件数、P1 件数、SLA 違反件数、平均解決時間を確認できます。ステータスボタンを押すと、その状態のインシデントだけを表示します。検索はタイトル、説明、起票者を対象にしており、優先度フィルタと組み合わせて使えます。

## シーン 4: インシデント起票（1分30秒）

### 実演

1. `インシデント起票` を押す。
2. タイトルと説明を入力する。
3. 優先度を P1 に変更する。
4. 担当者欄を確認または編集する。
5. `起票する` を押す。

### ナレーション

インシデント起票では、タイトルと説明が必須です。優先度は P3 が初期値で、担当者欄にはログインユーザー名が初期入力されます。起票すると `INC-0001` 形式の ID が採番され、SLA 期限が自動計算されます。Slack Webhook が設定されていれば、起票通知も送信されます。

## シーン 5: 詳細画面と更新（2分）

### 実演

1. 一覧の行をクリックして詳細へ移動する。
2. ヘッダーの優先度、ステータス、SLA バッジを示す。
3. 更新パネルでステータスを `in_progress` に変更する。
4. コメントを入力して `更新する` を押す。
5. タイムラインに変更履歴が追加されたことを示す。
6. クイック遷移ボタンで `resolved` へ進める。

### ナレーション

詳細画面では、インシデントの基本情報、SLA 期限、変更履歴を確認できます。更新パネルからステータス、優先度、担当者、コメントを保存できます。変更されたフィールドは監査ログとして記録され、コメントだけの更新もタイムラインに残ります。

ステータスが `new` 以外へ初めて変わると応答日時が記録され、`resolved` または `closed` へ変わると解決日時が記録されます。

## シーン 6: SLA 運用（1分）

### 実演

1. SLA バッジの残り時間を示す。
2. 応答済み、SLA 達成、期限超過の表示例を示す。
3. `/api/sla-check` の定期実行と `CRON_SECRET` 認証が必要であることを説明する。

### ナレーション

SLA は優先度ごとに応答期限と解決期限が決まっています。期限 30 分前から警告表示になり、期限を過ぎると違反表示になります。違反フラグは SLA チェック API が実行されたタイミングで更新されるため、本番環境では Vercel Cron などで `/api/sla-check` を定期実行します。本番では `CRON_SECRET` を設定し、Bearer 認証付きで呼び出します。

## シーン 7: admin の削除操作（1分）

### 実演

1. admin ユーザーでログインしている状態を示す。
2. 一覧のチェックボックスを選択する。
3. `N 件を削除` を押す。
4. `削除済みを表示` をオンにして削除済みだけを表示する。

### ナレーション

admin ロールでは、インシデントを一括で論理削除できます。削除されたインシデントは通常の一覧と統計から除外されます。必要に応じて、`削除済みを表示` をオンにすると削除済みだけを確認できます。現在の実装では復元機能はありません。

## シーン 8: まとめ（30秒）

### ナレーション

以上が ITSM Incident Tracker の基本操作です。日々の運用では、トップページで状況を把握し、必要なインシデントを詳細画面で更新し、SLA とタイムラインを確認しながら対応を進めます。
