# ユーザーマニュアル（member/admin）

## 1. ログイン

`/login` にアクセスします。

### パスワードログイン

1. ユーザー名を入力します。
2. パスワードを入力します。
3. `ログイン` を押します。
4. 成功すると元の画面、またはトップページへ移動します。

### GitHub ログイン

1. `GitHub でログイン` を押します。
2. GitHub の認可画面で承認します。
3. 初回ログイン時は `member` ロールのユーザーが自動作成されます。
4. 既存ユーザーとメールアドレスが一致する場合は GitHub アカウントが自動リンクされます。

### ログアウト

画面右上の `ログアウト` を押します。

## 2. トップページ

トップページ `/` には KPI、ステータス別件数、インシデント一覧が表示されます。

### KPI

| 表示 | 内容 |
|---|---|
| 総インシデント | 削除済みを除いた総件数 |
| 未解決 | `new + assigned + in_progress` |
| P1 Active | P1 優先度の件数 |
| SLA応答違反 | 応答期限を超過した件数 |
| SLA解決違反 | 解決期限を超過した件数 |
| 平均解決時間 | 解決済みインシデントの平均解決時間 |

### ステータスボタン

`new`、`assigned`、`in_progress`、`resolved`、`closed` のボタンを押すと、そのステータスで一覧を絞り込みます。同じボタンをもう一度押すと解除されます。

## 3. インシデント一覧

一覧には以下が表示されます。

| 項目 | 内容 |
|---|---|
| チェックボックス | admin の一括削除用 |
| ID | `INC-0001` 形式 |
| タイトル | インシデント概要 |
| 優先度 | P1 から P4 |
| ステータス | 現在の状態 |
| SLA（解決） | 解決期限の状態 |
| 起票者 | 起票したユーザー |
| 担当者 | 担当者名。未設定なら `-` |
| 起票日 | 起票日時 |

### 検索とフィルタ

- 検索ボックスはタイトル、説明、起票者を対象にします。
- 優先度は `すべての優先度`、`P1 Critical`、`P2 High`、`P3 Medium`、`P4 Low` から選びます。
- 更新アイコンを押すと一覧と KPI を再取得します。

### 削除済み表示と削除

admin の場合だけ以下を利用できます。

- `削除済みを表示`: 削除済みインシデントのみ表示します。
- チェックボックス: 削除対象を選択します。
- `N 件を削除`: 選択したインシデントを論理削除します。

削除済み行は薄く表示され、タイトルに取り消し線が付きます。削除済み行は詳細画面へ遷移しません。

## 4. インシデント起票

トップ右上の `インシデント起票` を押すとダイアログが開きます。

| 項目 | 必須 | 内容 |
|---|---|---|
| タイトル | 必須 | 短い概要 |
| 説明 | 必須 | 発生状況、影響範囲、暫定対処など |
| 優先度 | 任意 | 初期値は P3 |
| 担当者 | 任意 | 自由入力。初期値はログインユーザー名 |

`起票する` を押すとインシデントが作成され、一覧が更新されます。Slack Webhook が設定されている場合は通知が送信されます。

## 5. インシデント詳細

一覧の削除済みでない行をクリックすると詳細画面へ移動します。

### 表示内容

- ヘッダー: ID、優先度、ステータス、SLA 応答、SLA 解決
- タイトル: 鉛筆アイコンから編集可能
- 起票者と起票日時
- 説明
- SLA: 応答期限、応答日時、解決期限、解決日時
- タイムライン: 変更履歴とコメント
- 更新パネル: ステータス、優先度、担当者、コメント
- クイック遷移: 現在のステータスからよく使う遷移

### 更新方法

1. `更新者` を確認します。通常はログインユーザー名が自動入力されます。
2. ステータス、優先度、担当者、コメントを必要に応じて変更します。
3. `更新する` を押します。
4. 変更内容がタイムラインに記録されます。

コメントだけを入力して `更新する` を押した場合もタイムラインに記録されます。

### クイック遷移

| 現在 | ボタン |
|---|---|
| `new` | Assigned / In Progress / Closed |
| `assigned` | In Progress / Resolved / Closed |
| `in_progress` | Resolved / Closed |
| `resolved` | Closed / In Progress |
| `closed` | なし |

クイック遷移は操作補助です。通常のステータス選択では任意のステータスを選べます。

## 6. SLA の見方

| 表示例 | 意味 |
|---|---|
| `3h 20m` | 期限まで 3 時間 20 分 |
| `25m` | 期限まで 25 分 |
| `-40m` | 40 分超過 |
| `応答済` | 応答済み |
| `SLA達成` | 解決済み |

期限 30 分前から警告表示になり、期限超過で違反表示になります。トップページは 60 秒ごと、詳細画面は 30 秒ごとに表示が更新されます。

## 7. よくある質問

### ステータスを間違えて変更しました

詳細画面で正しいステータスへ更新してください。変更履歴はタイムラインに残ります。

### 担当者は一覧から選びますか

現在の実装では自由入力です。

### 削除したインシデントを見られますか

admin はトップページの `削除済みを表示` で確認できます。復元機能は現在ありません。

### SLA 違反は自動検知されますか

`/api/sla-check` が実行されたタイミングで違反フラグが更新されます。Vercel Cron などの定期実行設定と、本番環境では `CRON_SECRET` による認証設定が必要です。
