## ログイン

### パスワードログイン
1. `/login` ページにアクセス
2. **ユーザー名**を入力
3. **パスワード**を入力
4. **ログイン**ボタンをクリック
5. ダッシュボードに遷移します

### GitHub でログイン
1. `/login` ページの **GitHub でログイン** ボタンをクリック
2. GitHub 認可画面で **Authorize** をクリック
3. ユーザー情報が自動で紐付けられ、ダッシュボードに遷移します

**初回ログイン時は自動的に member ロールのユーザーが作成されます**

### ログアウト
1. 画面右上の **ユーザーメニュー** をクリック
2. **ログアウト** を選択
3. ログイン画面に戻ります

## ダッシュボード

ダッシュボード（`/`）は、システムの全体像を把握するための統計画面です。

### 表示される KPI

| KPI | 説明 |
|---|---|
| **総インシデント数** | 全期間のインシデント総数 |
| **オープン件数** | 未解決のインシデント数（new / assigned / in_progress） |
| **P1 アクティブ件数** | 最高優先度で未解決のインシデント数 |
| **SLA 応答違反件数** | 応答期限を超過したインシデント数 |
| **SLA 解決違反件数** | 解決期限を超過したインシデント数 |
| **平均解決時間** | インシデント起票から解決までの平均時間 |

### ステータス別フィルタ

KPI の下部に表示されるボタン：
- **new** — 未割当のインシデント
- **assigned** — 割当済み、未開始
- **in_progress** — 対応中
- **resolved** — 解決済み
- **closed** — クローズ済み

**ボタンをクリック**すると、インシデント一覧がそのステータスでフィルタされます。

## インシデント一覧

### アクセス方法
ダッシュボードの統計下部、またはサイドバーから **インシデント一覧** をクリック

### 表示項目

| 項目 | 説明 |
|---|---|
| **ID** | インシデント識別子（INC-0001 形式） |
| **タイトル** | インシデント概要 |
| **優先度** | P1 / P2 / P3 / P4（色付きバッジ） |
| **ステータス** | 現在の進捗状況 |
| **SLA（解決）** | 解決期限までの残り時間 |
| **起票者** | インシデントを報告したユーザー |
| **担当者** | 対応を担当するユーザー |
| **起票日** | インシデント起票日時 |

### フィルタリング機能

#### キーワード検索
1. 検索ボックスにテキストを入力
2. **タイトル・説明・起票者**から横断的に検索
3. リアルタイムで結果が絞り込まれます

#### 優先度フィルタ
- **全件** — フィルタなし
- **P1** — Critical（最高優先度）
- **P2** — High（高）
- **P3** — Medium（中）
- **P4** — Low（低）

#### ステータスボタン
- **new** — 未割当
- **assigned** — 割当済み
- **in_progress** — 対応中
- **resolved** — 解決済み
- **closed** — クローズ済み

### インシデント詳細へ移動
一覧の任意の行をクリックすると、インシデント詳細画面に遷移します。

## インシデント起票

### 画面へのアクセス
1. インシデント一覧ページ上部の **インシデント起票** ボタンをクリック
2. ダイアログが開きます

### 入力項目

#### タイトル（必須）
- インシデントの概要を 1 行で入力
- 例：「ログインシステムが応答しない」

#### 説明（必須）
- 詳細な状況、再現手順、影響範囲を記入
- 例：「午前 9:00 頃からログインが失敗するようになりました」

#### 優先度（任意）
デフォルト：P3

| 優先度 | 応答期限 | 解決期限 | 使用例 |
|---|---|---|---|
| **P1** | 15 分 | 4 時間 | システム全体停止 |
| **P2** | 60 分 | 8 時間 | 主機能が利用不可 |
| **P3** | 4 時間 | 24 時間 | 機能の一部が利用不可 |
| **P4** | 24 時間 | 72 時間 | 軽微な不具合 |

#### 担当者（任意）
- 即座に対応する担当者を指定します
- 未指定の場合、ステータスは「new」になります

### 起票実行
1. 必須項目（タイトル・説明）を入力
2. **起票** ボタンをクリック
3. インシデント ID が自動採番されます
4. **Slack 通知**が管理者に送信されます

### 起票後
- インシデント詳細画面に自動遷移
- インシデント ID（INC-0001 形式）を控えておく

## インシデント詳細・更新

### 画面構成

#### ヘッダー部分
- インシデント ID
- 優先度バッジ（色付き）
- ステータスバッジ
- SLA 応答状態バッジ
- SLA 解決状態バッジ

#### 基本情報
- **タイトル** — クリックで編集可能
- **起票者・起票日時**
- **説明文**

#### SLA 詳細
| 項目 | 説明 |
|---|---|
| **応答期限** | 最初に対応者が反応すべき期日時刻 |
| **応答日時** | 実際に対応が開始された日時 |
| **解決期限** | インシデントを解決すべき期日時刻 |
| **解決日時** | 実際に解決された日時 |

#### タイムライン
- 変更履歴とコメントが**新しい順**に表示
- 各エントリには操作者・操作内容・日時が記録

### 更新操作

#### ステータス変更
1. ステータスバッジをクリック
2. 遷移可能な次のステータスを選択
3. 変更完了後、Slack 通知が送信されます

**遷移ルール：**
- **new** → assigned / in_progress / closed
- **assigned** → in_progress / resolved / closed
- **in_progress** → resolved / closed
- **resolved** → closed / in_progress
- **closed** → （遷移不可）

#### 優先度変更
1. 優先度バッジをクリック
2. 新しい優先度（P1 ～ P4）を選択

#### 担当者変更
1. 現在の担当者名をクリック
2. 新しい担当者を選択

#### タイトル編集
1. タイトルをクリック
2. テキストを編集
3. **Enter** キーで保存

#### コメント追加
1. タイムライン下部のコメント入力ボックスに入力
2. **コメント送信** ボタンをクリック
3. タイムラインに記録されます

## SLA の見方

### SLA バッジの表示形式

| バッジ表示 | 意味 | 背景色 |
|---|---|---|
| **4h 30m** | 期限まで 4 時間 30 分 | 🟢 緑 |
| **20m** | 期限まで 20 分（警告） | 🟠 オレンジ |
| **-45m** | 既に 45 分超過（違反） | 🔴 赤 |
| **応答済** | 既に対応開始 | 🔵 青 |
| **SLA 達成** | 期限内に解決 | 🟢 緑 |

### 残り時間の更新
- バッジに表示される残り時間は **60 秒ごと**に自動更新されます
- ページを手動リロードしても更新されます

### SLA 違反時の対応

違反フラグが立つと：

1. **SLA 応答違反** — 応答期限を超過
   - ダッシュボード KPI に「SLA 応答違反件数」に加算
   - ステータス変更（new 以外）で回復

2. **SLA 解決違反** — 解決期限を超過
   - ダッシュボード KPI に「SLA 解決違反件数」に加算
   - ステータスを resolved / closed に変更で回復

3. **Slack アラート** — 違反検知時に Slack に通知

## よくある質問

### Q. インシデントのステータスを間違えて変更してしまった
**A.** タイムラインを確認し、前の状態に戻してください。変更履歴が記録されています。

### Q. 担当者が不在の場合、どうしたらいい？
**A.** 他の担当者に割り当て直してください。ステータスは自動的には変わりません。

### Q. SLA を超過してしまった場合、どうなる？
**A.** ダッシュボードに違反件数として集計されます。適切なステータス変更を行うと、一部フラグが解除されます。

### Q. 自分のインシデント一覧だけを見たい
**A.** キーワード検索で起票者またはユーザー名を入力してフィルタしてください。

### Q. パスワードを忘れた
**A.** ログイン画面で **GitHub でログイン** を使用するか、管理者に問い合わせてください。

### Q. GitHub アカウントとパスワードの両方でログインできる？
**A.** はい、メールアドレスが同じであれば自動的に紐付けされ、どちらでもログイン可能です。
